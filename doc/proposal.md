# 購物網站專案提案書 (Shopping Site Proposal)

## 1. 專案目標
本專案為一個「展示作品用 (Portfolio)」的標準 JavaScript 購物網站，旨在完整呈現從前端使用者介面到後端資料庫處理，以及金流與管理系統的整合開發能力。

## 2. 技術選型 (Tech Stack)
採用業界主流的 MERN 架構，並搭配最新版本的 Tailwind CSS 來建構：
*   **前端框架**: React
*   **後端框架**: Node.js (搭配 Express.js)
*   **資料庫**: MongoDB (NoSQL)
*   **樣式框架**: Tailwind CSS v4

## 3. 核心功能需求

### 3.1 前台 (一般使用者介面)
*   **完整的會員系統**: 
    *   使用者註冊、登入與登出 (預計使用 JWT 進行身份驗證)。
    *   個人資料管理與歷史訂單查詢。
*   **商品與購物流程**:
    *   商品列表展示、分類瀏覽與詳細資訊頁面。
    *   購物車功能 (加入商品、修改數量、移除商品)。
*   **結帳與多樣化金流系統**:
    *   **信用卡刷卡** (可串接 Stripe 或第三方金流測試環境)。
    *   **超商取貨付款** (實作台灣電商常見情境的流程與表單)。
    *   **虛擬貨幣支付 (ERC20)** (展示 Web3 整合能力，例如串接 MetaMask 錢包發送交易)。
*   **促銷機制**: 
    *   折扣碼 / 優惠折扣 (Discounts) 結帳時套用與金額計算。

### 3.2 後台 (管理員與員工系統)
*   **後台管理系統 (Admin Dashboard)**: 專屬的後台入口，需具備嚴格的權限控管 (RBAC)。
*   **員工管理**: 新增、刪除、停權及管理不同層級的後台員工帳號 (例如：超級管理員、一般員工)。
*   **商品與庫存管理**: 上架新商品、下架商品，編輯商品圖文資訊與價格。
*   **訂單管理**: 檢視所有使用者訂單，更新訂單狀態 (如：未處理、已出貨、已完成)。
*   **行銷與折扣管理**: 新增與管理折扣碼及促銷規則設定。

## 4. 系統架構簡述
本專案將採用**前後端分離**的設計模式：
*   **前端 (React)**：負責發送 API 請求、狀態管理及畫面渲染，提供使用者流暢的單頁應用程式 (SPA) 體驗。
*   **後端 (Node.js)**：提供 RESTful API 服務，負責處理所有業務邏輯、驗證權限，並與 MongoDB 進行資料讀寫。
